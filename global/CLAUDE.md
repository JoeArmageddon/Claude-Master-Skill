## Communication
- Be direct. No preamble, no filler ("Great question!", "Certainly!").
- State uncertainty explicitly. Never hallucinate file paths, API signatures, or library behaviour.
- When giving multiple options, include a clear recommendation.
- All code goes in code blocks — never embed snippets in prose.

## Workflow
- Run /start-session at the start of every new session before doing any work.
- Before multi-file changes, state the plan as a bullet list first.
- Ask before running destructive commands: `rm -rf`, `DROP TABLE`, `TRUNCATE`, `git reset --hard`.
- If a task is ambiguous, ask one clarifying question — not five.

## Code quality
- Explicit over implicit. Readable over clever.
- Leave `// TODO:` for known gaps — don't silently skip them.
- Never commit secrets, API keys, or credentials.
- Write error handling for every external call.

## Memory system
- SESSION_LOG.md, MEMORY.md, and PHASES.md are the source of truth. Never contradict them.
- Flag conflicts with MEMORY.md before proceeding — don't override silently.

## Token hygiene
- Use the Task tool for multi-file exploration. Return a summary, not raw content.
- Warn me at 70% context. Suggest /compact at 80%.
- Never embed full file contents in CLAUDE.md — use @path references.
